= Metadatos recopilados de fuentes de datos en NetApp Data Classification
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
NetApp Data Classification recopila metadatos mientras analiza tus fuentes de datos y sistemas. La mayoría de los metadatos están disponibles, pero algunos atributos no están disponibles para todos los protocolos.

[cols="15,25,25,25"]
|===
|  | *Metadatos* | *CIFS* | *No sé* 


.3+| *Marcas de tiempo* | _Hora de creación_ | Disponible | No disponible (no compatible con Linux) 


| _Última fecha de acceso_ | Disponible | Disponible 


| _Hora de la última modificación_ | Disponible | Disponible 


.2+| *Permisos* | _Permisos abiertos_ | Si el grupo "TODOS" tiene acceso al archivo, se considera "Abierto a la organización". | Si "Otros" tiene acceso al archivo, se considera "Abierto a la organización". 


| _Acceso de usuarios/grupos_ | La información de usuarios y grupos se toma de LDAP | No disponible (los usuarios NFS normalmente se administran localmente en el servidor, por lo tanto, el mismo individuo puede tener un UID diferente en cada servidor) 
|===
[NOTE]
====
* La clasificación de datos no extrae la "hora del último acceso" de las fuentes de datos de la base de datos.
* Las versiones anteriores del sistema operativo Windows (por ejemplo, Windows 7 y Windows 8) desactivan de forma predeterminada la recopilación del atributo «última hora de acceso» porque puede afectar el rendimiento del sistema. Cuando no se recopila este atributo, los análisis de Data Classification que se basan en la «última hora de acceso» se verán afectados. Puedes habilitar la recopilación de la última hora de acceso en estos sistemas Windows más antiguos si lo necesitas.


====


== Último tiempo de acceso

Cuando Data Classification extrae datos de recursos compartidos, el sistema operativo lo considera un evento de acceso y actualiza la "hora del último acceso" en consecuencia. Después de escanear, Data Classification intenta revertir la hora del último acceso a la marca de tiempo original. Si Data Classification no tiene permisos para escribir atributos en CIFS o permisos de escritura en NFS, el sistema no puede revertir la hora del último acceso a la marca de tiempo original.

De forma predeterminada, NetApp Data Classification no analiza los archivos cuando no dispone de permisos de escritura para restaurar la marca de tiempo original. Si tus datos admiten que el tiempo de último acceso no se restablezca a la marca de tiempo original, activa la opción *Analizar sin permisos de escritura* en la página de configuración del sistema.

image:screenshot_scan_missing_permissions.png["Una captura de pantalla que muestra el interruptor que puedes usar para que Data Classification analice tus archivos, incluso si no tiene los permisos correctos para revertir la última hora de acceso a la marca de tiempo original."]

Esta funcionalidad es aplicable a sistemas ONTAP locales, Cloud Volumes ONTAP, Azure NetApp Files, Amazon FSx for NetApp ONTAP y recursos compartidos de archivos de terceros.


NOTE: Los volúmenes ONTAP configurados con SnapLock tienen permisos de solo lectura y tampoco pueden restablecer la última hora de acceso a la marca de tiempo original.



=== Eventos de análisis de escaneo

En la página de Investigación hay un filtro llamado _Scan Analysis Event_ que te permite mostrar los archivos que no se clasificaron porque Data Classification no pudo revertir la hora de último acceso, o los archivos que sí se clasificaron aunque Data Classification no pudo revertir la hora de último acceso.

image:screenshot_scan_analysis_event_filter.png["Una captura de pantalla del filtro para ver qué archivos fueron escaneados y cuáles no, según tuvieran o no permisos de escritura."]

Las selecciones de filtro son:

* «Sin clasificar — No se puede restablecer el tiempo de último acceso» - Esto muestra los archivos que no se clasificaron debido a la falta de permisos de escritura.
* «Clasificados y con el tiempo de acceso actualizado»: muestra los archivos que se clasificaron y en los que Data Classification no pudo restablecer el tiempo de acceso a la fecha original. Este filtro solo es relevante en entornos donde esté habilitada la opción *Escanear sin permisos de escritura*.


Si es necesario, puedes exportar estos resultados a un informe para ver qué archivos se están analizando o no debido a los permisos. link:task-investigate-data.html#download-your-report["Obtenga más información sobre los informes de investigación de datos"^].
