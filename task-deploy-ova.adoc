= Implementa NetApp Data Classification con VMware Open Virtual Appliance
:allow-uri-read: 
:no-hardbreaks: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
NetApp Data Classification admite la implementación con VMware Open Virtual Appliance (OVA), proporcionando un paquete gestionado para instalar y ejecutar Data Classification en entornos VMware vSphere locales. El método de implementación OVA proporciona un entorno gestionado y prevalidado por NetApp para simplificar la implementación.

Este método se recomienda para las nuevas implementaciones basadas en VMware.



== Prerrequisitos

Antes de utilizar el archivo OVA para instalar Data Classification, debes tener:

* Un entorno VMware vSphere
* Una dirección IPv4 estática asignada al appliance
* Servidores DNS accesibles desde la red de destino
* Acceso SSH habilitado al dispositivo
* link:https://mysupport.netapp.com/site/user/registration["NetApp credenciales del sitio de soporte"] para descargar el instalador




== Implementa el OVA en VMware vSphere

. Descarga el archivo OVA de link:https://mysupport.netapp.com/site/products/all/details/cloud-data-sense/downloads-tab/["Sitio de soporte de NetApp"^]. Elige el dispositivo OVA adecuado para tu entorno:
+
|===
| Appliance OVA | Tamaño de producción | Límite de exploración de archivos 


| `DataClassification-<version>-L.ova` | Carga de trabajo de producción estándar: 16 CPU, 64 GB de RAM, 500 GB de disco | 250 millones de archivos 


| `DataClassification-<version>-XL.ova` | Gran carga de trabajo de producción: 32 CPU, 128 GB RAM, 1 TB disco | 500 millones de archivos 
|===
. Introduce el cliente vSphere. Haz clic con el botón derecho en el clúster o host de destino y selecciona *Deploy OVF Template* > *Create VM from OVA*.
. Sube el archivo OVA que descargaste en el primer paso.
. Especifica el *nombre de la máquina virtual* y el centro de datos, host o clúster y almacén de datos de destino.
. Acepta el Acuerdo de licencia de usuario final de NetApp.
. Personaliza la plantilla. Debes proporcionar la siguiente información:
+
** **Hostname** - el nombre del appliance
** **Dirección IPv4** - la dirección IPv4 estática para la interfaz de gestión
** **Máscara de subred IPv4** - por ejemplo 255.255.255.0
** **Dirección de la pasarela IPv4** - la pasarela por defecto
** **DNS primario** - el resolvedor de DNS requerido
+
Opcionalmente, añade el **DNS secundario** y, como listas separadas por comas, los **dominios de búsqueda** (por ejemplo, `corp.local,lab.local`) y los **servidores NTP**.



. Establece la contraseña *Admin* para el usuario Linux predeterminado.
+

NOTE: La contraseña se utiliza para el acceso a raíz, el acceso SSH y la administración del sistema. Guarda la contraseña de forma segura.

. Revisa el resumen. Selecciona *Finalizar* para desplegar.
. Una vez completado el despliegue, enciende la máquina virtual.




== Instala clasificación de datos

. Descarga el software de clasificación de datos desde la link:https://mysupport.netapp.com/site/products/all/details/cloud-data-sense/downloads-tab/["Sitio de soporte de NetApp"^].
+
Selecciona el archivo llamado *DATASENSE-INSTALLER-<version>.tar.gz*.

. En la consola, seleccione *Gobernanza > Clasificación*.
. Selecciona *Desplegar clasificación*.
+
image:screenshot-deploy-classification.png["Una captura de pantalla de la selección del botón para activar la Clasificación de datos."]

. Dependiendo de si está instalando Data Classification en una instancia que preparó en la nube o en una instancia que preparó en sus instalaciones, seleccione la opción *Implementar* adecuada para iniciar la instalación de Data Classification.
. Se muestra el cuadro de diálogo _Implementar clasificación de datos en las instalaciones_.  Copie el comando proporcionado (por ejemplo: `sudo ./install.sh -a 12345 -c 27AG75 -t 2198qq` ) y pégalo en un archivo de texto para que puedas usarlo más tarde.  Luego seleccione *Cerrar* para cerrar el cuadro de diálogo.
. Conéctate mediante SSH al dispositivo OVA usando el usuario administrador.
. Ejecuta el comando de instalación que generaste para completar la instalación y registrar el sistema con tu organización.
